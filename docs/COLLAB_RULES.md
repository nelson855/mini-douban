# 协作规则（长期生效）

1. 命令执行者
- 由你（用户）在本地终端执行所有命令。
- 我（Codex）默认不主动执行命令。

2. 每次代码变更后的交付方式
- 我会提供一份清晰、简短的执行步骤（命令 + 预期结果 + 下一步）。
- 说明聚焦“做什么”和“做完看什么”，不写冗长背景。

3. 输出载体
- 我会把需要执行的步骤写入 `docs/RUN_STEPS.md`。
- `docs/RUN_STEPS.md` 采用“版本化追加记录”，不覆盖历史。
- 版本规则：最新版本放在最上面（倒序）。
- 若本轮没有新增执行步骤或步骤无变化，则不新增记录，避免重复。

4. 异常处理
- 如果你执行命令报错，我会基于报错信息给出下一步排查命令和修复建议。
